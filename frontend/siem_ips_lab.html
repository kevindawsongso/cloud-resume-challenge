SIEM/IPS/IR Lab

Project Breakdown: VMWare Workstation Cyber Lab

Project Goal

This virtual cyber lab was built with the dual objectives of designing and administering an operational cybersecurity environment to execute, detect, log, monitor, and remediate cyber threats[cite: 21], and strengthening skills in incident response, penetration testing, threat intelligence, and security administration[cite: 30].

Implemented Technologies

  • Network Operations: PF Sense Firewall/Router, Windows Defender Firewall, and domain controller based DHCP/DNS were implemented for comprehensive network operations[cite: 22, 31].
  • Security Operations: Splunk ES SIEM[cite: 23, 32], Suricata IDS/IPS[cite: 23], Security Onion intrusion prevention/intrusion detection[cite: 32], and Tenable Nessus Vulnerability Scanner [cite: 23, 32] were deployed for security monitoring and operations purposes.

Lab Hosts and Red Team Efforts

The lab hosts comprised of Windows Enterprise, Ubuntu Server, and Kali Linux machines. Kali Linux was specifically utilized for red team efforts, including enumeration and exploitation of network resources, as well as malware delivery[cite: 24, 33].

Telemetry Collection and Analysis

Telemetry was collected via Splunk Universal Forwarders[cite: 25, 34]. For in-depth analysis, data was aggregated at a search head, utilizing Splunk Search Processing Language[cite: 25].

Remediation Actions

In response to configured alerts and logs, remediation actions were completed by hardening exposed network services, patching affected systems, or managing user accounts[cite: 26, 35].

Outcomes & Learnings

This lab provided hands-on experience in:

  • Designing and administering an operational cybersecurity environment[cite: 21].
  • Executing, detecting, logging, monitoring, and remediating cyber threats[cite: 21].
  • Strengthening skills in incident response, penetration testing, threat intelligence, and security administration[cite: 30].
  • Practical application of firewalls, SIEM, IDS/IPS, and vulnerability scanning tools.
  • Understanding red team methodologies and their impact on network resources.
